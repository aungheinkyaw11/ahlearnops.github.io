Kubernetes LoadBalancer service on Amazon EKS

A step-by-step guide to Kubernetes LoadBalancer service on Amazon EKS.

This note demonstrates how to set up a Kubernetes LoadBalancer service on Amazon EKS, expose it externally, and secure it with HTTPS using AWS Certificate Manager and Route 53.

Prerequisites#

  • AWS CLI configured with appropriate permissions
  • eksctl installed
  • kubectl installed and configured

Step 1: Create an EKS Cluster#

First, create a test EKS cluster using eksctl:

eksctl create cluster \
  --name test-eks \
  --region ap-south-1 \
  --version 1.35 \
  --instance-types t3.small \
  --nodes 1 \
  --nodes-min 1 \
  --nodes-max 1

This command creates a single-node EKS cluster in the ap-south-1 region.

Step 2: Deploy Sample Applications#

We’ll deploy two sample applications: a counting service and a dashboard that consumes it.

Counting Service#

Create counting.yaml:

apiVersion: v1
kind: ServiceAccount
metadata:
  name: counting
---
apiVersion: v1
kind: Service
metadata:
  name: counting
spec:
  ports:
  - port: 9001
    targetPort: 9001
  selector:
    app: counting
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: counting
spec:
  replicas: 1
  selector:
    matchLabels:
      app: counting
  template:
    metadata:
      labels:
        app: counting
    spec:
      containers:
      - name: counting
        image: hashicorp/counting-service:0.0.2
        ports:
        - containerPort: 9001

Apply it:

kubectl apply -f counting.yaml

Dashboard Service#

Create dashboard.yaml:

apiVersion: v1
kind: ServiceAccount
metadata:
  name: dashboard
---
apiVersion: v1
kind: Service
metadata:
  name: dashboard
spec:
  type: LoadBalancer
  selector:
    app: dashboard
  ports:
  - port: 80
    targetPort: 9002
---
apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    app: dashboard
  name: dashboard
spec:
  replicas: 1
  selector:
    matchLabels:
      app: dashboard
  template:
    metadata:
      annotations:
      labels:
        app: dashboard
    spec:
      serviceAccountName: dashboard
      containers:
      - name: dashboard
        image: hashicorp/dashboard-service:0.0.4
        ports:
        - containerPort: 9002
        env:
        - name: COUNTING_SERVICE_URL
          value: 'http://counting.default.svc.cluster.local:9001'

Apply it:

kubectl apply -f dashboard.yaml

Step 3: Verify Services#

Check the services:

kubectl get svc

Output:

NAME         TYPE           CLUSTER-IP      EXTERNAL-IP                                                               PORT(S)        AGE
counting     ClusterIP      10.100.149.59   <none>                                                                    9001/TCP       7m25s
dashboard    LoadBalancer   10.100.89.242   a9dd99b117d0c4a35b11a12334f42ac6-409196634.ap-south-1.elb.amazonaws.com   80:30582/TCP   7m24s
kubernetes   ClusterIP      10.100.0.1      <none>                                                                    443/TCP        26m

The dashboard service has an external IP provided by AWS Classic Load Balancer.

Step 4: Access via HTTP#

Initially, the service is accessible via HTTP on port 80.

Step 5: Add DNS with Route 53#

Create a CNAME record in Route 53 pointing to the Load Balancer DNS name.

Domain: dashboard.learnops.site

Step 6: Configure HTTPS#

Add SSL Certificate#

Create a public certificate in AWS Certificate Manager for your domain.

Update Load Balancer Listener#

Modify the Load Balancer to use HTTPS on port 443, attaching the certificate.
